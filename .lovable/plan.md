

# Cash Offer Options Page Audit + Chip Loop Fix

## Problems Identified

### 1. Chip Loop: "Estimate my net proceeds" persists after completion
The user ran the calculator, then opened Selena — and Selena kept showing "Estimate my net proceeds" as a chip. This happens because:

- The chip governance on the **server** checks `tools_completed` from the context passed by the client
- But the client sends context **at message time**, and if the chat was opened before the calculator was completed, the stale context carries forward
- The client-side `chipGovernance.ts` has its own `TOOL_SUPPRESSED_CHIPS` map, but it only filters chips when the context is refreshed

**Root cause**: `SelenaChatContext` captures `tools_completed` at chat-open time. If the user opens chat from the hero, runs the calculator mid-page, then returns to chat — the session context sent to the server is stale.

**Fix**: In `SelenaChatContext.tsx`, refresh `tools_completed` from `getSessionContext()` at **every message send**, not just at chat-open. This ensures the server always gets the latest completion state.

### 2. "Compare Your Options" page is not dynamic or personalized
The results page shows the same static output regardless of user context:
- No acknowledgment of returning users who already ran it
- No personalization based on their motivation/timeline selection
- The "Cost of Time" section uses generic labels ("Traditional Advantage" / "Cash Advantage") without connecting to the user's stated situation
- The recommendation reason is generated by the algorithm but doesn't reference their motivation

**Fix**: Add a single personalized insight line in `CalculatorResults.tsx` that references the user's selected motivation and timeline, making the data feel custom. Example: "Because you're dealing with a life change and need to move within 30 days, here's how the numbers compare."

### 3. Page structure has redundant static content below the dynamic calculator
After the calculator results (which are dynamic and personalized), the page shows:
- "Quick Reference: Cash vs. Traditional" — a **static** pros/cons grid that **duplicates** what the calculator results already show
- "When Cash Makes Sense" / "Traditional May Be Better" — **static** checklist that doesn't reflect the user's actual inputs

These sections add scroll depth without adding value. A user who just ran their numbers doesn't need a generic static comparison below it.

**Fix**: Remove the static "Quick Reference" section entirely (lines 101-192 of V2CashOfferOptions.tsx). Keep the "Watch Out for Wholesalers" warning (unique, high-value content). Condense "When Cash Makes Sense" into the calculator's Step 3 (Next Steps) as a contextual callout based on their recommendation, not a static grid.

### 4. Terminal CTA doesn't connect to the journey
The bottom navy CTA says "Getting clearer on your home's value?" — generic. It should reflect that they just completed the calculator and offer a **specific** next step.

**Fix**: Make terminal CTA dynamic based on whether calculator was completed. If completed: "You've seen the numbers — let's talk about what fits your situation." If not completed: keep current copy.
